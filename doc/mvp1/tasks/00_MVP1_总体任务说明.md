## 1. 工作目标

MVP-1 的目标是在 MVP-0 已冻结的 fake 纵向链路基础上，把任务从
`mvp0_home` 扩展为一个完整但仍然纯软件的单臂 Pick-and-Place 闭环。

本阶段要证明的是：

```text
一条 StartTask 请求
-> 场景重置
-> 机械臂移动到抓取相关目标
-> 夹爪化手部开合
-> 物体逻辑附着
-> 搬运到放置区域
-> 物体解除附着
-> 任务成功或失败可观测
```

MVP-1 仍然不证明真实抓取、真实接触或真实装配能力。它验证的是任务分层、
状态机、fake 模块协作、错误码、日志和自动化测试。

## 2. 本阶段范围

新增或扩展以下 fake 能力：

```text
fake_arm_control:
  支持多个命名目标

fake_hand_control:
  提供夹爪化的 open / preshape / close / hold / release 能力

fake_scene_manager:
  维护物体位置、附着状态和场景重置状态

fake_terminal_operation:
  提供 PLACE 末端操作占位能力

assembly_bringup:
  提供 MVP-1 一键启动 launch 文件

assembly_tests:
  提供 MVP-1 集成验收测试
```

推荐新增或修改：

```text
doc/mvp1/tasks/
src/assembly_interfaces/
src/assembly_task/
src/fake_arm_control/
src/fake_hand_control/
src/fake_scene_manager/

```

## 3. 非目标

MVP-1 暂不实现：

```text
真实 Franka Research 3 控制
真实因时 RH56DFX-2R 全关节控制
MoveIt / MTC 真实规划
MuJoCo 或其他物理仿真
视觉定位
力控制或触觉闭环
真实接触和摩擦
Peg-in-Hole 插入
双臂协同
任务取消、暂停、恢复
多任务并发
自动失败恢复
学习模型或抓取生成模型
```

## 4. 固定任务定义

MVP-1 推荐保留 MVP-0 的 `mvp0_home`，同时新增：

```text
task_name = "pick_and_place_mvp"
arm_name = "right_arm"
hand_name = "right_hand"
object_id = "mvp_object"
source_location = "pickup_zone"
target_location = "place_zone"
terminal_operation = "PLACE"
```

`right_arm` 仍逻辑映射到首个或右侧 Franka Research 3。`right_hand` 仍逻辑
映射到首个或右侧因时 RH56DFX-2R，但在 MVP-1 中只按夹爪或预设抓型执行器
处理。

## 5. 推荐状态机

成功路径：

```text
IDLE
-> RESETTING
-> ARM_HOME
-> HAND_OPEN
-> ARM_PREGRASP
-> HAND_PRESHAPE
-> ARM_GRASP
-> HAND_CLOSE
-> ATTACH_OBJECT
-> ARM_LIFT
-> ARM_PREPLACE
-> ARM_PLACE
-> DETACH_OBJECT
-> HAND_OPEN
-> ARM_RETREAT
-> ARM_HOME
-> SUCCESS
```

失败路径：

```text
任一关键步骤失败 -> FAILED
```

每个状态都应发布 `TaskState`，并带上：

```text
task_id
current_state
previous_state
progress
error_code
message
```

## 6. 接口规划

### 6.1 StartTask

沿用 MVP-0 的 `/assembly/start_task` 和 `StartTask.srv`。MVP-1 暂不要求
StartTask 请求中携带复杂参数；任务参数可以先来自固定配置。

要求：

```text
task_name = "mvp0_home"            # 继续可用
task_name = "pick_and_place_mvp"   # MVP-1 新增
```

### 6.2 MoveArm

沿用 `/assembly/move_arm` 和 `MoveArm.action`，扩展支持的 `target_name`：

```text
home
pregrasp
grasp
lift
preplace
place
retreat
```

MVP-1 中这些目标仍然是 fake 命名目标，不代表真实轨迹规划。

### 6.3 Hand Control

推荐新增一个手部控制接口，形式可以是 service 或 action。MVP-1 优先选择
service，保持实现简单：

```text
/assembly/control_hand
```

建议请求字段：

```text
string hand_name
string command       # open / preshape / close / hold / release
string grasp_name    # default / pinch / power，可先固定为 default
float32 timeout_sec
---
bool success
int32 error_code
string message
```

### 6.4 Scene Object State

推荐扩展 `fake_scene_manager`，维护 object 的 fake 状态：

```text
object_exists
object_attached
attached_link
object_location
```

推荐新增接口：

```text
/assembly/get_object_pose
/assembly/attach_object
/assembly/detach_object
```

如果希望减少接口数量，也可以先合并为一个通用 object state service，但接口
命名必须清楚，测试必须覆盖 attach / detach 后的状态变化。

### 6.5 Terminal Operation

推荐新增：

```text
/assembly/execute_terminal_operation
```

MVP-1 只支持：

```text
operation_type = "PLACE"
```

`INSERT`、对孔、柔顺控制和力反馈放到 Post-MVP。

### 任务 2：扩展 Fake Scene Manager

目标：



目标：



### 任务 4：实现 Fake Terminal Operation

```text

```

### 任务 5：扩展 Fake Arm Control

目标：

```text

```

```text

不支持的 target_name 返回 2002
timeout_sec 非法返回 2003

```

### 任务 6：扩展 Assembly Task 状态机

目标：

```text

按 MVP-1 状态机顺序调用 scene / arm / hand / terminal operation
每一步发布 TaskState
```

验收：

```text

```

## 9. 完成定义

MVP-1 完成不等于“多写几个 fake 节点”，而是满足：

```text
1. 可以从干净环境 colcon build；
2. 一条 launch 命令启动完整 MVP-1 fake 系统；
3. StartTask(pick_and_place_mvp) 能触发完整任务；
4. 可以观察到完整 Pick-and-Place 状态序列；
5. 物体 attach / detach 状态可被验证；
6. 非法请求有明确错误码；
7. 关键失败点进入 FAILED；
8. 连续 10 次执行不崩溃、不死锁；
9. mvp0_home 回归仍通过；
10. 验收文档和 Git tag 可以定位冻结版本。
```

推荐版本标签：

```text
mvp1.0.0
```

## 10. 推荐开发顺序

推荐顺序：

```text
1. 冻结接口和配置
2. 扩展 fake_arm_control 命名目标
3. 实现 fake_hand_control
4. 扩展 fake_scene_manager 物体状态
5. 实现 fake_terminal_operation
6. 扩展 assembly_task 状态机
7. 增加 MVP-1 launch
8. 增加 MVP-1 集成测试
9. 做干净环境回归
10. 冻结 MVP-1 文档和 tag
```

实现时应优先保持 MVP-0 回归测试通过；每完成一个 fake 模块，就补对应的
最小测试或集成检查，避免最后一次性排查整条链路。
