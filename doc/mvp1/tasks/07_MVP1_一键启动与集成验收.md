# MVP-1 第七阶段：一键启动与集成验收

## 1. 工作目标

为 MVP-1 提供一条 launch 命令和一组自动化集成测试，证明完整 Fake
Pick-and-Place 链路可以重复运行。

## 2. 本阶段范围

涉及软件包：

```text
assembly_bringup
assembly_tests
assembly_interfaces
assembly_task
fake_scene_manager
fake_arm_control
fake_hand_control
fake_terminal_operation
```

推荐新增：

```text
assembly_bringup/launch/mvp1_fake_pick_place.launch.py
assembly_tests/test/test_mvp1_pick_place_flow.py
```

## 3. Launch 要求

一条命令启动：

```bash
ros2 launch assembly_bringup mvp1_fake_pick_place.launch.py
```

应启动：

```text
fake_scene_manager_node
fake_arm_control_node
fake_hand_control_node
fake_terminal_operation_node
assembly_task_node
```

启动后应能发现：

```text
/assembly/start_task
/assembly/reset_scene
/assembly/move_arm
/assembly/task_state
/assembly/control_hand
/assembly/get_object_pose
/assembly/attach_object
/assembly/detach_object
/assembly/execute_terminal_operation
```

## 4. 集成测试覆盖

MVP-1 集成测试至少覆盖：

```text
合法 pick_and_place_mvp 请求
空 task_name
不支持的 task_name
完整成功状态序列
物体 attach / detach 状态变化
连续 10 次合法任务
task_id 不重复且递增
mvp0_home 回归
```

建议逐步增加失败链路：

```text
不支持的 hand command
不支持的 arm target
不支持的 terminal operation
attach / detach 前置状态错误
依赖服务不可用
```

## 5. 状态序列断言

成功路径至少断言以下顺序：

```text
RESETTING
ARM_HOME
HAND_OPEN
ARM_PREGRASP
HAND_PRESHAPE
ARM_GRASP
HAND_CLOSE
ATTACH_OBJECT
ARM_LIFT
ARM_PREPLACE
ARM_PLACE
TERMINAL_PLACE
DETACH_OBJECT
HAND_OPEN
ARM_RETREAT
ARM_HOME
SUCCESS
```

如果最终实现选择不同状态名，必须在任务文档和接口快照中同步冻结。

## 6. 测试实现建议

沿用 MVP-0 测试风格：

```text
测试自己启动 launch
测试创建 ROS node
等待 service/action/topic 可用
调用 StartTask
订阅 TaskState
按 task_id 过滤状态
使用 timeout 防止无限等待
失败时输出 launch 日志尾部
```

cd /home/ace/bimanual_dexterous_mvp_ws

## 8. 完成标准

```text
MVP-1 launch 可一键启动
MVP-1 集成测试通过
MVP-0 mvp0_home 回归通过
连续 10 次 pick_and_place_mvp 不崩溃、不死锁
失败时日志和 TaskState 可定位失败步骤
```
