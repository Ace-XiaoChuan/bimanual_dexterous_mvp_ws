# MVP-1 第一阶段：接口与配置冻结

## 1. 工作目标

本阶段先冻结 MVP-1 的接口增量、固定任务参数和错误码范围，避免后续 fake
模块各自定义一套字段和语义。

完成后应明确：

```text
Pick-and-Place 固定任务名
手部控制接口
物体状态接口
末端操作接口
机械臂命名目标集合
错误码范围
配置文件位置和字段
```

assembly_interfaces

## 3. 固定任务参数与配置字段

MVP-1 固定任务参数以以下配置文件为冻结来源：

```text
src/assembly_task/config/mvp1_pick_and_place.yaml
```

该配置只服务 fake 单臂 Pick-and-Place 闭环，不表达真实轨迹、真实抓取位姿或
真机控制参数。任务编排节点后续应从该文件读取固定任务名、逻辑对象、命名目标、
手部命令、抓型和超时时间。

### 3.1 任务级参数

MVP-1 新增任务：

```text
task_name = "pick_and_place_mvp"
arm_name = "right_arm"
hand_name = "right_hand"
object_id = "mvp_object"
source_location = "pickup_zone"
target_location = "place_zone"
terminal_operation = "PLACE"
```

MVP-0 任务必须继续保留：

```text
task_name = "mvp0_home"
```

### 3.2 机械臂命名目标

MVP-1 冻结以下 fake 机械臂命名目标：

```text
home
pregrasp
grasp
lift
preplace
place
retreat
```

这些目标只表示 fake_arm_control 支持的高层目标名，不表示 MoveIt 轨迹、
真实关节角或笛卡尔位姿。

### 3.3 手部命令

MVP-1 冻结以下 fake 手部高层命令：

```text
open
preshape
close
hold
release
```

这些命令只表达夹爪化或预设抓型语义，不表示 RH56DFX-2R 全关节精细控制。

### 3.4 抓型参数

MVP-1 默认抓型先固定为：

```text
grasp_name = "default"
```

后续可扩展 `pinch`、`power` 等抓型，但第一阶段先冻结字段位置和默认值。

### 3.5 超时参数

MVP-1 冻结以下超时字段，单位为秒：

```text
move_arm_sec = 5.0
control_hand_sec = 3.0
scene_operation_sec = 2.0
terminal_operation_sec = 3.0
```

任务编排节点后续调用 action 或 service 超时后，应进入 `FAILED`，并通过
`TaskState` 发布对应错误码和 message。

## 4. 接口草案

### ControlHand.srv

```text
string hand_name
string command
string grasp_name
float32 timeout_sec
---
bool success
int32 error_code
string message
```

`command` 支持：

```text
open
preshape
close
hold
release
```

### AttachObject.srv

```text
string task_id
string object_id
string link_name
---
bool success
int32 error_code
string message
```

### DetachObject.srv

```text
string task_id
string object_id
string target_location
---
bool success
int32 error_code
string message
```

### GetObjectPose.srv

MVP-1 可以先不返回真实几何 pose，只返回 fake 状态和 location：

```text
string object_id
---
bool success
string object_location
bool object_attached
string attached_link
int32 error_code
string message
```

### ExecuteTerminalOperation.srv

```text
string task_id
string operation_type
string object_id
string target_location
float32 timeout_sec
---
bool success
int32 error_code
string message
```

MVP-1 只支持：

```text
operation_type = "PLACE"
```

## 5. 错误码范围

建议冻结以下范围：

```text
4001  HandControl: 不支持的 hand_name
4002  HandControl: 不支持的 command
4003  HandControl: timeout_sec 非法

5001  SceneManager: object_id 不能为空
5002  SceneManager: object 不存在
5003  SceneManager: object 已附着
5004  SceneManager: object 未附着

6001  TerminalOperation: 不支持的 operation_type
6002  TerminalOperation: target_location 不能为空

7001  AssemblyTask: hand control 不可用或无响应
7002  AssemblyTask: scene object 操作不可用或无响应
7003  AssemblyTask: terminal operation 不可用或无响应
```

本阶段只冻结接口和配置，不实现完整业务行为。fake 节点可以先不接入新接口，
但接口一旦冻结，后续阶段不应随意改字段名和语义。
