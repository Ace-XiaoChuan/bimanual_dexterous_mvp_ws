# MVP-0 第二阶段工作说明：首批最小接口

## 1. 工作目标

在 `assembly_interfaces` 软件包中实现第一批 ROS 2 自定义接口，并完成接口生成和编译验证。

本阶段只建立通信契约，不实现状态机、客户端或 Fake Server。

## 2. 本阶段接口范围

只实现以下 4 个接口：

```text
msg/TaskState.msg
srv/ResetScene.srv
srv/StartTask.srv
action/MoveArm.action
```

暂时不实现：

```text
HandCommand.action
TerminalOperation.action
AttachObject.srv
DetachObject.srv
StopTask.srv
ResetTask.srv
GetTaskState.srv
```

这些接口将在第一条最小链路运行成功后逐步添加。

## 3. 各接口职责

### TaskState.msg

用于发布任务状态，包括：

* 任务编号；
* 当前状态；
* 上一个状态；
* 执行进度；
* 错误码；
  -状态说明。

### ResetScene.srv

用于请求 Fake Scene Manager 重置场景。

响应需要明确表示：

* 是否成功；
* 错误码；
* 状态说明。

### StartTask.srv

用于启动一次新的任务。

响应需要明确表示：

* 是否接受启动请求；
* 新任务的 `task_id`；
* 错误码；
* 状态说明。

### MoveArm.action

用于请求指定机械臂移动到一个命名目标。
当前硬件基线中，`right_arm` 逻辑名映射到首个或右侧 Franka Research 3。

第一阶段只需要支持：

```text
arm_name
target_name
timeout_sec
```

Action 应返回：

* 是否成功；
* 错误码；
* 状态说明。

Action 执行期间应反馈：

* 当前状态；
* 执行进度。

## 4. 工程修改内容

需要在 `assembly_interfaces` 中创建：

```text
assembly_interfaces/
├── action/
│   └── MoveArm.action
├── msg/
│   └── TaskState.msg
├── srv/
│   ├── ResetScene.srv
│   └── StartTask.srv
├── CMakeLists.txt
└── package.xml
```

需要修改：

* `CMakeLists.txt`；
* `package.xml`；
* `rosidl_generate_interfaces` 配置；
* 接口生成相关依赖。

## 5. 验收标准

以下检查必须全部通过。

### 编译接口包

```bash
cd ~/bimanual_dexterous_mvp_ws

source /opt/ros/$ROS_DISTRO/setup.bash

colcon build \
  --packages-select assembly_interfaces \
  --symlink-install
```

### 加载工作区

```bash
source install/setup.bash
```

### 查看生成的接口

```bash
ros2 interface show assembly_interfaces/msg/TaskState
ros2 interface show assembly_interfaces/srv/ResetScene
ros2 interface show assembly_interfaces/srv/StartTask
ros2 interface show assembly_interfaces/action/MoveArm
```

验收要求：

* `assembly_interfaces` 编译成功；
* 4 个接口均能通过 `ros2 interface show` 查看；
* 没有缺少依赖或接口生成错误；
* 其他 7 个软件包仍能正常编译；
* 不包含任何节点实现；
* 不依赖 MoveIt、MuJoCo、Franka Research 3 或因时 RH56DFX-2R 真实硬件。

## 6. Git 提交
