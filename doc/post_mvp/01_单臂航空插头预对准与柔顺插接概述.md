# Post-MVP 1 概述：单臂航空插头预对准与低速柔顺插接

## 1. 研究定位

MVP-2 和 MVP-3 分别完成 MoveIt 规划链路与 MuJoCo 物理仿真链路的
Pick-and-Place 验证。Post-MVP 1 不再重复验证物体能否被搬运，而是进入与项目
目标直接相关的接触型装配问题：航空插头能否在受控误差下完成稳定插接。

本轮的核心研究问题是：

```text
给定插头和插座存在小的初始位置、姿态误差，单臂 FR3 能否先到达预插接位姿，
再通过低速轴向运动、接触状态和虚拟柔顺策略完成插接，并在卡滞时安全退出？
```

## 2. 最小实验边界

```text
平台：MuJoCo 单 FR3 + 夹爪化 / 预设抓型 RH56
对象：connector plug 与固定 connector socket
位姿来源：第一轮使用 MuJoCo ground truth
任务：pre_insert -> low_speed_insert -> success / abort
运动：固定插接轴、低速直线接近、有限搜索或退出
控制：位置控制基础上的虚拟柔顺、阻抗或 admittance 策略
```

第一轮不引入视觉、双臂、多指精细操作或真机。这些能力会改变失败来源，无法
判断结果是由坐标系误差、接触参数、控制策略还是感知误差造成。

## 3. 先决条件

在开始插接研究前，应满足：

```text
MVP-3 的 MuJoCo 场景、/clock、关节控制和 object state 同步稳定
FR3 可在仿真中执行固定 pre_insert 目标
plug 可通过确定性 grasp-hold 状态被携带和释放
connector plug / socket mesh 的尺度和碰撞开口已检查
```

插头和插座的 STL 几何中心不等同于功能装配坐标系。必须先定义：

```text
plug_frame             插头功能参考系
socket_frame           插座功能参考系
insertion_axis         插接正方向
pre_insert_pose        接触前的安全目标位姿
target_insertion_depth 目标插入深度
```

## 4. 建议状态机

```text
PRE_INSERT
-> ALIGN
-> APPROACH
-> CONTACT_DETECTED
-> COMPLIANT_INSERT
-> INSERTED
-> VERIFY
-> SUCCESS

任一异常
-> STOP
-> RETRACT
-> FAILED
```

`CONNECTOR_INSERT` 应作为独立 terminal operation，而不是复用 `PLACE` 的成功
语义。每个阶段应发布 TaskState，并记录插接深度、接触状态、控制模式、错误码
和退出原因。

## 5. 分步研究计划

### 5.1 几何与轨迹基线

```text
已知精确 plug/socket pose
从 pre_insert_pose 沿 insertion_axis 低速直线运动
无初始误差或大间隙场景
记录深度、最终相对位姿和碰撞状态
```

目标是证明功能坐标系、插接轴和低速轨迹定义正确，不把这一步包装成柔顺控制
成果。

### 5.2 接触与虚拟柔顺

```text
读取仿真接触、末端力或等价观测
设置轴向与横向不同的刚度 / 阻尼
接触后限制速度并允许小幅横向补偿
超过力、姿态、时间或偏移阈值时 stop + retract
```

该步骤比较刚性位置控制和虚拟柔顺策略在相同初始误差下的表现。

### 5.3 误差鲁棒性评估

对插头相对插座施加受控扰动：

```text
横向位置误差
轴向初始距离
绕插接轴和垂直轴的姿态误差
接触摩擦、刚度和阻尼变化
```

每组条件多次运行，记录成功、超时、卡滞、超力和主动退出。

## 6. 评价指标

```text
insertion_success_rate  插接成功率
max_contact_force       最大接触力或等价接触指标
insertion_time          从 approach 到 verify 的耗时
final_depth             最终插入深度
final_pose_error        最终 plug/socket 相对位姿误差
abort_rate              主动退出率
jam_rate                卡滞率
recovery_success_rate   retract 后 reset / 重试成功率
```

研究结论应基于误差扫描下的指标变化，而不是只展示一次无误差成功动画。

## 7. 技术选择原则

```text
MoveGroupInterface：用于到达 pre_insert 等无接触目标
MuJoCo：用于接触、物理状态和控制策略比较
PlanningScene：用于规划碰撞镜像与 RViz 观察
MTC：可在后续用于多阶段几何可行解，不是本轮接触控制的前置条件
视觉：下一轮作为 pose-provider，将 ground truth 替换为有噪声估计
```

MTC 不能替代接触后的柔顺、卡滞检测或失败恢复。因此在当前固定单臂任务中，
先保留状态机和 MGI 路径，将研究资源集中在插接控制闭环。

## 8. 后续迭代顺序

```text
Post-MVP 1  单臂、仿真真值位姿、预对准与柔顺插接
Post-MVP 2  加入有噪声 pose-provider / 视觉，评估感知误差鲁棒性
Post-MVP 3  多指抓型、抓取稳定性或双臂辅助定位
Post-MVP 4  在明确安全边界下开展真机低风险接触实验
```

双臂和多指灵巧手应建立在单臂插接基线之上。否则无法区分新增自由度带来的
收益与系统复杂度造成的失败。

## 9. 非目标

```text
不在本轮宣称完成真实航空插头装配
不要求真实 FR3 / RH56 真机控制
不做全空间视觉定位、手眼标定或大范围搜索
不做学习策略、模仿学习或强化学习
不以一次成功演示代替可重复的实验评估
```
