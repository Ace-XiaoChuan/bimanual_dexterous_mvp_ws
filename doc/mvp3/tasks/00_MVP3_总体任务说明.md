## 1. 工作目标

MVP-3 在 MVP-2 的单臂 MoveIt Pick-and-Place 基础上，引入 MuJoCo 作为物理
仿真后端，验证仿真关节控制、仿真时间、场景重置、物体状态同步和基础接触
参数。RH56DFX-2R 在本阶段仍按夹爪化或预设抓型执行器使用。

本阶段要证明的是一条可重复的仿真闭环：

```text
StartTask
-> MoveArm 规划结果驱动 MuJoCo FR3 关节
-> Hand preset 驱动仿真手部预设状态
-> connector plug 的抓取保持状态可观测
-> 物体被放到 place_zone
-> ResetScene 可恢复确定初始状态
```

## 2. 总体架构

```text
assembly_task
-> MoveArm.action / ControlHand / AttachObject / DetachObject
-> MoveIt arm adapter + MuJoCo execution adapter
-> ROS 2 - MuJoCo bridge
-> MuJoCo model, controller, clock, joint/object state

moveit_scene_manager
<-> PlanningScene mirror

MuJoCo scene
-> FR3 + 夹爪化 RH56 + table + connector plug + connector socket
```

MuJoCo 是仿真中关节和物体状态的事实来源。PlanningScene 仍可用于规划和 RViz
展示，但不能用逻辑附着状态覆盖 MuJoCo 已观测到的对象状态。

## 3. 阶段范围

```text
单 FR3
夹爪化或预设抓型 RH56
MuJoCo 仿真时间与关节控制
connector plug / socket 场景资产
固定 Pick-and-Place
确定性 grasp-hold 约束
ROS 2 状态、错误码、日志和测试
```

## 4. 非目标

```text
真实 FR3 FCI 或真机控制
RH56 多指精细控制、触觉或滑移检测
自动接触抓取判定
socket 对准、插接、CONNECTOR_INSERT
力控、阻抗控制、柔顺控制
视觉定位、手眼标定
双臂协同
MTC 任务构造
```

## 5. 完成标准

```text
MuJoCo 场景可加载并以固定 timestep 稳定运行
ROS 2 可读取 /clock、joint state 和对象状态
MoveArm 轨迹可驱动仿真 FR3 到达目标
ResetScene 可恢复关节、手部和物体初始状态
完整 StartTask 可在 MuJoCo 中完成固定 Pick-and-Place
主要失败路径进入 FAILED，且状态可观测
headless 测试、运行记录和已知限制写入 doc/mvp3/acceptance/
```
