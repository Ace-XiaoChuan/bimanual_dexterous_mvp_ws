# MVP-3 第九阶段：收尾冻结与验收证据

## 1. 工作目标

在 MVP-3 功能完成后，冻结可复现的 MuJoCo 仿真基线，记录模型来源、构建环境、
测试结果、任务演示和已知限制。

## 2. 建议证据目录

```text
doc/mvp3/acceptance/
├── mvp3_completion_checklist.md
├── model_and_asset_result.md
├── bridge_and_clock_result.md
├── build_and_test_result.md
├── interface_snapshot.md
├── mujoco_pick_and_place_demo_result.md
└── known_limitations.md
```

## 3. 必须记录的事实

```text
Git commit hash 和依赖版本
MuJoCo version、模型来源、mesh 路径和缩放
FR3 / hand / plug / socket 的模型边界
构建、headless 测试和启动命令
仿真时间、joint state、object state 和 reset 证据
StartTask 成功路径和主要失败路径
已知的不稳定接触、约束和坐标系限制
```

## 4. 冻结验收标准

```text
干净环境可构建相关 package
MuJoCo 模型可 headless 加载
bridge、轨迹、场景同步和 task 编排测试通过
固定 Pick-and-Place 演示可复现
连续运行与 reset 无状态泄漏
README 只保留入口与当前阶段，详细证据位于 acceptance 目录
```

## 5. 冻结后非目标说明

冻结文档必须明确 MVP-3 不代表：

```text
真实 FR3 或 RH56 真机控制
多指精细操作或可靠物理抓取
接触力闭环或柔顺控制
航空插头对准、插接和 CONNECTOR_INSERT
双臂协同
视觉定位和手眼标定
```

## 6. 版本标记

在全部验收记录完成后，再单独创建 MVP-3 冻结提交和 tag。tag 名称、提交范围和
是否推送必须由当时实际验收状态决定，不能在本任务文档阶段预先宣称完成。
