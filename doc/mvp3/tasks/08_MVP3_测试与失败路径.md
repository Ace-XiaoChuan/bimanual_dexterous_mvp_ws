# MVP-3 第八阶段：测试与失败路径

## 1. 工作目标

为 MuJoCo 模型、bridge、轨迹执行、物体同步和任务编排建立 headless 优先的
测试与失败路径证据。

## 2. 测试层次

```text
模型层：MJCF 加载、body/joint 名称、keyframe、固定步进
bridge 层：/clock、joint state、TF、reset、命令缓冲
执行层：轨迹下发、qpos 误差、超时、controller unavailable
场景层：plug pose、grasp-hold、detach、连续 reset
编排层：StartTask 成功、FAILED、重复执行、资源清理
```

headless 测试应使用固定 timestep、固定随机种子（若使用）和明确超时，避免依赖
RViz 或人工点击。

## 3. 最少失败覆盖

```text
MJCF 或 mesh 路径无效
MuJoCo model load 失败
bridge 未启动或 /clock 停滞
joint state 不完整
MoveArm 目标非法、规划失败、执行超时
close 后不满足 grasp-hold 前置条件
重复 Attach、未 Attach Detach
reset 后 object state 未恢复
执行中再次 StartTask
```

## 4. 验收标准

```text
相关 package build 通过
模型和 bridge 的 headless 测试可重复运行
主要成功与失败路径均有断言
测试清理后无残留 MuJoCo、ROS 2 或 launch 子进程
测试结果包含命令、环境、时间和失败原因
```

## 5. 非目标

```text
不要求物理参数与真机完全一致
不把 GUI 可视化作为自动化测试前置条件
```
