# MVP-3 第六阶段：夹爪化 RH56 与确定性抓取约束

## 1. 工作目标

让 `ControlHand` 命令驱动 MuJoCo 中夹爪化或预设抓型的 RH56 状态，并提供一个
可重复的 grasp-hold 约束，使 Pick-and-Place 任务能在物理仿真中携带 plug。

## 2. 手部范围

```text
open / preshape / close / hold / release
每个 command 映射到一组固定 hand qpos 或 actuator target
手部状态和命令结果可观测
```

RH56 模型可以用于可视化和基础碰撞，但本阶段不以多指接触稳定性作为成功判据。

## 3. 抓取约束语义

首轮允许采用确定性 grasp-hold constraint，例如在已到达固定 grasp pose 且收到
close 命令后创建 plug 与 `fr3_hand` 的固定约束；release 或 reset 时删除约束。

```text
close 成功 != 自动检测真实抓取成功
grasp-hold active = 任务层可依赖的确定性携带状态
接触信息只用于观察和调参，不作为本阶段唯一抓取判据
```

约束创建前必须校验 plug 处在允许抓取区域；约束创建失败要返回错误而不是将
对象状态直接标记为 attached。

## 4. 验收标准

```text
ControlHand 的 open / close / release 可改变仿真手部状态
允许抓取区域内 close 后可创建 grasp-hold constraint
携带期间 plug 相对 fr3_hand 的变换保持稳定
release / reset 后 constraint 被删除，plug 回到物理 world state
不满足抓取前置条件时 AttachObject 明确失败
```

## 5. 非目标

```text
不做触觉闭环、滑移检测或自动抓取质量评分
不做 RH56 全关节抓型生成
不做复杂摩擦参数辨识
```
