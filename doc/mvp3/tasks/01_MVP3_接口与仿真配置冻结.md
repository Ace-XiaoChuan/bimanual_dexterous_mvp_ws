# MVP-3 第一阶段：接口与仿真配置冻结

## 1. 工作目标

冻结 MVP-3 的仿真后端配置、名称域和接口边界，使 MuJoCo 替换执行后端时不
破坏 MVP-2 已稳定的任务接口。

## 2. 保持稳定的上层接口

```text
/assembly/start_task       StartTask.srv
/assembly/move_arm         MoveArm.action
/assembly/control_hand     ControlHand.srv
/assembly/reset_scene      ResetScene.srv
/assembly/attach_object    AttachObject.srv
/assembly/detach_object    DetachObject.srv
/assembly/task_state       TaskState.msg
```

`assembly_task` 不直接调用 MuJoCo Python/C API；仿真后端由专用 bridge 或
execution adapter 封装。

## 3. 建议冻结配置

建议新增 `mvp3_mujoco_pick_and_place.yaml`，至少包含：

```text
task_name: pick_and_place_mvp
sim_backend: mujoco
planning_group: fr3_arm
arm_name: right_arm
hand_mode: gripper_preset
object_id: mvp_object
object_model: connector_plug
source_location: pickup_zone
target_location: place_zone
sim_timestep_sec
control_rate_hz
scene_operation_sec
planning_timeout_sec
execution_timeout_sec
```

`right_arm` 是任务层逻辑名，`fr3_arm` 是 MoveIt 规划组，`fr3_hand` 是 MuJoCo
和 MoveIt 模型使用的具体末端 link；三者不得混用。

## 4. 错误码原则

```text
保持既有 1000/2000/3000/5000/7000 范围语义
新增仿真错误使用独立范围，例如 8000
模型加载、bridge 不可用、时钟停滞、控制超时必须可区分
```

## 5. 验收标准

```text
配置可被解析并安装到 package share 目录
接口字段和 MVP-2 一致
任务名、规划组、link 名和模型名有明确映射
错误码表和超时语义被记录
```

## 6. 非目标

```text
不在本阶段创建 MuJoCo 模型
不修改已有 .srv / .action 结构
不加入插接或力控参数
```
