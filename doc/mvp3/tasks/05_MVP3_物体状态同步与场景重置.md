# MVP-3 第五阶段：物体状态同步与场景重置

## 1. 工作目标

使 connector plug 的 MuJoCo pose、附着状态和 reset 结果可被 ROS 2 与
PlanningScene 一致观察。MuJoCo 是物体物理状态的事实来源。

## 2. Reset 语义

```text
恢复 FR3 home qpos
恢复手部预设 open qpos
恢复 plug 到 pickup_pose
清除 grasp-hold constraint
恢复 socket、table 和 fixture 的固定状态
发布新的 joint state、object state 和 PlanningScene mirror
```

重复 reset 必须得到同一状态，不能积累 collision object、constraint 或缓存命令。

## 3. 场景同步原则

```text
MuJoCo plug pose -> ROS object state -> PlanningScene world object
MuJoCo grasp-hold active -> ROS attached state -> PlanningScene attached object
MuJoCo reset / release -> 清除 attached object 并恢复 world object
```

同步应带有时间戳或 sequence，避免旧状态覆盖 reset 后的新状态。规划镜像更新
失败必须可报告，不能静默把 MuJoCo 与 RViz 显示为不同状态。

## 4. 验收标准

```text
reset 后 plug 位姿与配置一致
物体 world / attached 状态可通过 ROS 接口查询
RViz 中对象状态与 MuJoCo 状态一致
连续 reset 不产生重复对象或残留约束
```

## 5. 非目标

```text
不从视觉估计物体 pose
不做 socket 插接成功判断
不把 PlanningScene 当作物理引擎
```
