# MVP-3 第四阶段：仿真机械臂轨迹执行

## 1. 工作目标

使 MVP-2 的 `MoveArm.action` 在保持上层语义不变的条件下，规划并驱动 MuJoCo
中的 FR3 关节执行轨迹。

## 2. 职责边界

```text
assembly_task
  只发送 MoveArm goal 和处理结果

moveit_arm_control
  使用 MoveGroupInterface 进行目标解析和规划

MuJoCo execution adapter
  接收规划轨迹，按仿真时间下发并观测 joint state

MuJoCo bridge
  把控制命令作用到执行器并发布状态
```

不得让 `assembly_task` 直接写 MuJoCo qpos，也不得让 MoveIt fake-hardware
controller 与 MuJoCo 同时控制同一组 FR3 关节。

## 3. 执行要求

```text
支持 home / pregrasp / grasp / lift / preplace / place / retreat 的固定任务目标
轨迹时间与 /clock 对齐
执行开始、进行、完成、超时和失败可返回给 MoveArm.action
执行后检查关节位置误差，不只检查命令已发送
```

## 4. 验收标准

```text
MoveArm(right_arm, home) 可改变 MuJoCo FR3 qpos
至少一个 pick-and-place 相关目标可规划并执行
非法 target 返回既有明确错误
bridge 不可用、轨迹超时和执行误差过大时返回可定位错误
RViz joint state 与 MuJoCo qpos 一致
```

## 5. 非目标

```text
不增加 MTC
不做在线轨迹重规划
不做真机轨迹下发
```
